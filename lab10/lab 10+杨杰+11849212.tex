\section{\textbf{Introduction}}
Thresholding may be viewed as a statistical-decision theory problem whose
objective is to minimize the average error incurred in assigning pixels to two
or more groups. There are several methods be used wildly in this area, some of them are base on region, some are base on intensity.

In this experiment, Otus’s method will be used on one noisy image. And using Partition Otus’s method to segment an ununiform illuminate image. After that, moving average thresholding used to segment handwriting image. Finally  region growing method used to perform segmentation on two images. Through this experiment, students will know how to use these methods in image segmentation and choose parameters.




\section{\textbf{Method}}
\subsection{Otus's method}
Otsu's method (Otsu [1979]), is
an attractive alternative. The method is optimum in the sense that it maximizes
the  \textsl{between-class\ variance}, a well-known measure used in statistical discriminant analysis. The basic idea is that well-thresholded classes should be distinct
with respect to the intensity values of their pixels and, conversely, that a threshold giving the best separation between classes in terms of their intensity values
would be the best (optimum) threshold. In addition to its optimality, Otsu's
method has the important property that it is based entirely on computations
performed on the histogram of an image, an easily obtainable 1-D array
\begin{algorithm}[H]
\caption{Otus's method}
\label{alg:Otus}
\begin{algorithmic}[1] %每行显示行号
\Require image $A$,
\Ensure segmented image $S$, threshold T
\State mean value  $ A \to M$
\For{each  $k$ \textbf{in} $[0,L-1]$}
    \State Counter number of $A == k \to num$
    \State background pixel number $wB += num$
    \State Pixel number of $A - wB \to $ foreground pixel number $wF$
    \State mean value of  background $\to sumB + num*k$
    \State mean value of foreground $\to (M*A-sumB)/WF$

    \If{wB == 0 \textbf{or} wF == 0}
        \State \textbf{continue}
    \EndIf
    \State $\sigma=wF*wB*(M-mF)*(M-mB)$
    \If {$\sigma > \sigma_{max}$}
        \State Threshold $T \gets k$
        \State $\sigma_{max} \gets \sigma$
    \EndIf
\EndFor
\State $S \gets A \geq T$

\end{algorithmic}
\end{algorithm}

Otsu's method traverse all of intensity value for $0$ to $L-1 (255)$. For each intensity, use it as threshold to divide image into two group, for each group, counter pixel number and mean weighted value \eref{eq:otus}. The calculate the between-class variance $\sigma$ base on \eref{eq:sig}, compare $sigma$ to $sigma_max$, to select best threshold $T$ which let $\sigma$ has maximum value \eref{eq:otsuT}.
The whole algorithm of Otsu's method is listed in \aref{alg:Otus}.

\begin{equation}\label{eq:otus}
\begin{array}{lcr}
  P_k = \sum\limits_{i\in C_k} p_i&\omega_1 = \sum_{k=0}^l P_k &\omega_2 = \sum_{k=l+1}^{L-1} P_k\\
  \mu_1=\frac{1}{\omega_1}\sum_{k=0}^l kP_k&
  \mu_2=\frac{1}{\omega_2}\sum_{k=l+1}^{L-1} kP_k&
  \mu_T=\frac{1}{\omega_1+\omega_2}\sum_{k=0  }^{L-1} kP_k \\
\end{array}
\end{equation}

\begin{equation}\label{eq:sig}
  \sigma_B^2=\omega_1(\mu_1-\mu_T)^2+\omega_2(\mu_2-\mu_T)^2=\omega_1\omega_2(\mu_1-\mu_2)^2\\
\end{equation}

\begin{equation}\label{eq:otsuT}
  T = \mathop{argmax}\limits_{k\in \{0,\dots,  L-1\}}\frac{\sigma_B^2}{\sigma_B^T}=\mathop{argmax}\limits_{k\in \{0,\dots,  L-1\}}\sigma_B^2
\end{equation}




Partition Otsu's method is a kind of modified Otsu's method, which subdivide an image into small nonoverlapping rectangles so that the illumination
of each rectangle is uniform. The method compensates the non-uniformities in illumination and/or reflectance. The relation ship between Partition Otsu's method and Otsu's method like global thresholding and local thresholding. 



\subsection{Moving average thresholding}
Moving average thresholding is a kind of sliding window method. During scanning, pixels are visited in a zigzag path, so we compute average of the intensity of last $n$ visited pixels \eref{eq:mov}. The segmentation can be
performed using a threshold $T_xy = bm_xy$, $b$ can be a suitable constant such
as $b=0.5$. Such method is best used for document thresholding. A rule of thumb
is to let $  n$ equal $5$ times the average stroke width. 
\begin{algorithm}[H]
\caption{Region growing segmentation of defective weld X-ray image}
\label{alg:mov}
\begin{algorithmic}[1] %每行显示行号
\Require image $f$, stroke size $n$
\Ensure segmented image $S$
\For{each double row line \textbf{in} $A$}
    \State reverse this line
\EndFor
\State 1 D vector $D \gets$ stretch reversed image to 1 row
\State zero padding vector $PD \gets$ pad $n$ 0s at head of $D$
\For{x \textbf{in} range(\textsl{len}(D))}
    \State $T[x]=\frac{1}{n}\sum\limits_{i=x-1-n}^x PD_i$
\EndFor
\State thresholding 1 D vector $TD \gets D \geq T$
\For{each double row line \textbf{in} $TD$}
    \State reverse this line
\EndFor
\State  segmented image $S \gets$ double line reversed $TD$
\end{algorithmic}
\end{algorithm}

The whole algorithm of moving average thresholding is listed in \aref{alg:mov}

\begin{equation}\label{eq:mov}
\begin{split}
  m(k+1) =& \frac{1}{n}\sum^{k+1}_{k+2-n}z_i\\
  =&m(k)+\frac{1}{n}(z_{k+1}-z_{k+1-n})
\end{split}
\end{equation}





\subsection{Region growing segmentation}
The goal of region growing segmentation is group pixels or sub-regions into larger regions based on predefined
criteria for growth. It start with a set of “seed” points, then grow regions by appending to each seed those neighboring pixels that have predefined properties similar to the seed (e.g., gray level, texture…). The selection of the seeds can be operated manually or using automatic
procedures based on a-priori knowledge can be included and  strictly application-dependent. The growing is controlled by the connectivity. The stop rule is another parameter of the algorithm. It can depend on the apriori knowledge on the problem or geometric structures. 

\begin{algorithm}[H]
\caption{Region growing segmentation of defective weld X-ray image}
\label{alg:xray}
\begin{algorithmic}[1] %每行显示行号
\Require image $f$,
\Ensure segmented image $Seg$
\State initial seed img $S \gets f > 245$
\State growing region $Q \gets |f-S|<=68$
\State seed img $Sd \gets $ erose $S$ to each connection part only 1 pixel
\State segmented image $Seg \gets$ connection analysis$(Q,Sd)$
\end{algorithmic}
\end{algorithm}
Because the image of this experiment used in this part has different property. One is in noisy image while another can be classified by intensity. They have different seeds selection method and criteria. Their specific algorithms are listed in \aref{alg:xray} and \aref{alg:noise}, respectively.

\begin{algorithm}[H]
\caption{Region growing segmentation of noisy region image}
\label{alg:noise}
\begin{algorithmic}[1] %每行显示行号
\Require image $f$,
\Ensure segmented image $Seg$
\State blur img $B \gets $ cv2.blur$(f)$
\State initial seed img $S \gets |f - 127| > 10$
\State seed img $Sd \gets $ erose $S$ to each connection part only 1 pixel
\State growing region $Q \gets |B- 127| > 1$
\State segmented image $Seg \gets$ connection analysis$(Q,Sd)$
\end{algorithmic}
\end{algorithm}

\section{\textbf{Discussion}}
\fref{fig:otsu} indicated that single threshold Otus's method has low robustness, it is very easy be influenced by noisy. The mean filter is necessary before using Otus's method. Otus;s method has perfect performance when image only has less than two peaks.

When the illumination of image is not uniform, partition Otus's method can be used. So that in each sub image, the intensity variance will decrease, which let it can be segmented by Otus'   s method respectively. 

From \fref{fig:subhist} can see that the second column sub images has overlap between two Gaussian cluster. Because of this overlap, Threshold $T$ is in polygon cluster, which induce classify several low intensity pixels as background. This is the origin of noise. To avoid this, original image can be divided into more sub images. But more partition require more calculation. Besides this,  too much partition may remove the peaks of sub image, which lead to bad performance. This is a tradeoff.

It is of interest to note that successful segmentation results were obtained in
both cases using the same parameters which shows the relative ruggedness of the approach. In general, thresholding based on moving averages
works well when the objects of interest are small (or thin) with respect to the
image size, a condition satisfied by images of typed or handwritten text.

The region growing segmentation can be used to detect interested region precisely. Its seed is very to determine, they are like the high threshold in Canny while the second requirement is hard to determine, it like the low threshold in Canny. They both have two
screening steps.
